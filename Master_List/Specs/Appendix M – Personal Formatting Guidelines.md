# Appendix M – Personal Formatting Guidelines

This appendix is the personal formatting reference for all Master List Framework documents. It captures the formatting decisions used throughout the framework so future documents remain consistent without redefining styles in each section.

# **M.1 Page Layout**

* Platform: Google Docs (primary authoring).  
* Page Size: Letter (8.5 × 11 in).  
* Margins: 1 inch on all sides.  
* Alignment: Left.  
* First-line indent: None.  
* Body paragraphs use consistent spacing rather than blank lines.  
* Heading 1 and every Appendix begin on a new page.

# **M.3 Tables**

* Use native Google Docs tables.  
* Header fill: \#A2BDDC.  
* Header font: Calibri Bold, color \#366091.  
* Body font: Cambria 11 pt.  
* 0.5 pt black borders.  
* Top vertical alignment.  
* Repeat header row when exporting to Word.  
* Keep tables left aligned.

# **M.4 Lists**

* Bullets use standard round bullets.  
* Numbered lists preserve authored numbering.  
* Nested lists use consistent indentation.  
* Use lists for procedures and requirements.

# **M.6 Document Standards**

* Include a Table of Contents for major specifications.  
* Use Heading styles instead of manual formatting.  
* Use tables where they improve readability.  
* Keep diagrams centered with descriptive titles.  
* Appendices begin on a new page and use consistent numbering.

# **M.7 Personal Preferences**

Favor clean, professional layouts over decorative formatting. Consistency is more important than visual complexity. Documents should be easy to read in Google Docs and export cleanly to DOCX and PDF without requiring manual reformatting.
