## 5. Call Graph Summary

- `onOpen` builds the Master List Tools menu and exposes monthly update, Demo P, monthly reports, template, Dashboard Quality, index, archive, and configuration workflows.
- `runMonthlyUpdate` calls monthly change report build, Demo P sync, disenrolled list update, Master List creation, index rebuild, and tab ordering.
- `createMasterList` prompts for month and delegates to `createMasterListForMonth_`, which locates Demo P, optionally deletes an existing Master List, creates the new output, copies/syncs data, formats, hides templates, and writes timing.
- `doGet` is a web-app restore endpoint and delegates to active-index or archive-workbook restore functions under a document lock.
- Dashboard Quality workflows write staged or immediate section blocks to the Dashboard Quality sheet/properties.

## 6. Public Library API Report

Supported public API should be limited to documented menu/admin/workflow entry points: `onOpen`, `onEdit`, `doGet`, monthly workflow functions, formatter functions, template/dashboard setup functions, index restore/configuration functions, and validation/diagnostic runners. All underscore-suffixed helpers should be treated as internal, but because Apps Script libraries expose top-level functions, current internals remain accidental public surface until a compatibility plan is approved.

## 7. Orphan and Duplicate Code Report

- No confirmed duplicate top-level production function names were found by the function declaration scan. The apparent duplicates are nested local sort helpers (`name_`, `dateRank_`) in separate functions.
- Locally duplicated nested helper names: `name_` and `dateRank_` appear in separate sort functions; this is lower risk but hurts traceability.
- No-direct-caller functions detected: 122. These are not safe removal candidates without external Apps Script/library/menu/string trigger verification.

## 8. Performance Report

- Destructive/write/format/sort/copy operation hits identified by static search: 240. Major hotspots are template generation, Dashboard Quality section replacement, monthly output creation, Master List build, and archive/restore.
- Prefer one read/in-memory transform/one write for each workflow, cache dashboard config/header maps/sheet refs, batch format row/column runs, and avoid repeated shell rebuilds when signatures match.

## 10. Trigger and Concurrency Report

- Simple triggers/web handlers found: `onOpen`, `onEdit`, `doGet`. Menu-triggered manual workflows are numerous.
- `runWithWorkflowBusyFlag_` uses `LockService.getDocumentLock()` and a document-property busy flag, but not every public destructive workflow is visibly routed through it.
- Minimum safe strategy: all workflows that create/delete/copy/rename/clear output sheets should use the document lock plus a `try/finally` busy flag and should not expose active-sheet destructive helpers without validation.

## 11. Error Handling, Logging, Security, and Authorization

- Errors are frequently logged via best-effort warning helpers; this is good for resilience but can hide diagnostic write failures.
- `doGet` needs HTML escaping for request and error data.
- OAuth scopes include spreadsheets, container UI, scriptapp, and script storage. No URL fetch/mail/drive scopes were required by static service scan.
- Document properties store archive spreadsheet ID, timing flag, template signatures, index restore URL, and Dashboard Quality buffers; document these as deployment state.
